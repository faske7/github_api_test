# GitHub Repository Manager

Этот скрипт на Python взаимодействует с GitHub API для создания, проверки и удаления репозиториев. Он использует библиотеки `requests` и `python-dotenv` для работы с API и управления переменными окружения.

## Функционал

- **Создание репозитория**: Создает новый репозиторий на вашем GitHub аккаунте.
- **Проверка списка репозиториев**: Проверяет наличие созданного репозитория в списке всех репозиториев.
- **Удаление репозитория**: Удаляет репозиторий по его имени.

## Установка

1. **Клонируйте репозиторий**:

    ```bash
    git clone https://github.com/faske7/github_api_test.git
    ```

2. **Перейдите в каталог проекта**:

    ```bash
    cd github_api_test
    ```

3. **Установите зависимости**:

```bash
pip install -r requirements.txt
```

4. **Создайте файл `.env`** в корневом каталоге проекта и добавьте в него следующие переменные:

    ```env
    API_KEY=your_personal_access_token
    USERNAME=your_github_username
    ```

    Замените `your_personal_access_token` и `your_github_username` на ваши реальные данные.

## Использование

1. **Запуск скрипта**:

    ```bash
    python test_api.py
    ```

    Скрипт выполнит следующие действия:
    - Создаст новый репозиторий с именем, указанным в переменной `new_repo`.
    - Проверит, существует ли созданный репозиторий в списке ваших репозиториев.
    - Удалит репозиторий с именем, указанным в переменной `new_repo`.

2. **Файлы журнала**:

    - Логи скрипта записываются в файл `app.log` в корневом каталоге проекта.

## Примечания

- Убедитесь, что ваш токен имеет необходимые разрешения для создания и удаления репозиториев.
- Скрипт предполагает, что переменные окружения загружаются из файла `.env`, который не должен попадать в систему контроля версий. Убедитесь, что вы добавили `.env` в `.gitignore`.
